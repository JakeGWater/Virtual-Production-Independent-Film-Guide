Contributing¶

Thank you for contributing! We have tried to organize the site such that we can include a variety of content.

Overview¶

  1. Open a pull-request on GitHub with any content, even an empty file. This gives us a place to begin the conversation.

  2. Work with the maintainers to craft your content for the site.

  3. Try to follow the Guidelines as much as possible.

  4. Once approved, your content will be merged and immediately visible.

Guidelines¶

Contributions whenever possible should adhere to the following guidelines:

  1. Contributions must be centred around virtual production.

  2. Contributions should try to fit into the existing pattern. As a workflow, reference, studio setup, or resource.

  3. Contributions must be Production Ready or clearly marked as Experimental.

  4. Contribution which are not complete must be marked as Work in Progress.

  5. External Links:

    1. Must go in the relevant References section. Use a Citation to cross-link from elsewhere.

    2. Exception studio pages (Studios) may use external links, including affiliate links.

  6. Embedding content, such as videos, figures, slides is encouraged and permitted. The content origin must be on our list of Allowed Embedding Domains.

  7. Images must be hosted from vpifg.com

  8. Use References for External Content

Allowed Embedding Domains¶

  • youtube.com

  • docs.google.com

  • figma.com

Work in Progress¶

Any incomplete content must be marked as a work in progress with the following display:

Work In Progress

This page is a Work in Progress and may be incomplete.

Insert the following code immediately after the page title.

.. wip::

If content is also experimental, it should be marked as both.

Experimental¶

We encourage experimental setups, but we don’t want to send people down the wrong path. Experimental pages should begin with the following display:

Work In Progress

This page is Experimental and may not be Production Ready.

Insert the following code into the beginning of your page:

.. experimental::

References for External Content¶

All references to external content should use citations whenever possible.

For example:

  1. Refer to to the citation from your guide or workflow as follows:

    Use the [unrealengine]_ to ...

  2. Cite the content under the relevant References file as:

    .. [unrealengine] The unreal game engine `<https://www.unrealengine.com>`_

Using citations keeps things under control, and ensures that all links are reusable and relevant.
